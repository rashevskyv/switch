### Проблеми з запуском прошивки

{% spoiler Приставка зависає на логотипі Switch %}

Можливі шляхи вирішення проблеми:

1. Переконайтеся, що можете завантажитися в сток. Якщо це так, можливо ви встановили якісь системні модулі, які заважають завантаженню. Для цього: 
	1. Запустіть {% include abbr/uk/hekate.txt abbr="hekate" %} -> **Payloads** -> `TegraExplorer` -> `kefir-helper.te`
	1. Якщо після цього приставка запрацювала, шукайте проблему в установлених модулях 
1. Перевстановіть {% include abbr/uk/kefir_addr.txt %} начисто. Для цього видаліть з карти пам'яті все, крім папки `Nintendo` та `emummc` і ставте кефір заново
1. Відформатуйте вашу карту пам'яті в [FAT32](https://format.customfw.xyz){:target="_blank"}. 
1. Перевстановіть {% include abbr/uk/kefir_addr.txt %} начисто. Для цього видаліть з карти пам'яті все, крім папки `Nintendo` та `emummc` (або `sxos/emunand`) і ставте кефір заново
1. Перевірте чи монтується розділ `USER`. Для цього: 
	1. Запустіть {% include abbr/uk/hekate.txt abbr="hekate" %} -> **Payloads** -> `TegraExplorer.bin`
	1. Виберіть **Browse EMUMMC** якщо не запускається {% include abbr/uk/emunand.md abbr="EmuNAND" %} або **Browse EMMC**, якщо не запускається {% include abbr/uk/sysnand.md abbr="SysNAND" %}
	1. Перейдіть у розділ **USER**. Якщо помилок немає, то розділ USER монтується і проблема не в ньому
1. [Відновіть бекап](/backup-nand#відновлення-резервної-копії){:target="_blank"}, якщо такий є
1. Зробіть [скидання до заводських налаштувань](/system-wipe){:target="_blank"}
1. Зробіть [відновлення прошивки](/downgrade_fw){:target="_blank"}

{% endspoiler %}

{% spoiler При запуску приставки нічого не відбувається / Не можу ввімкнути приставку %}

Можливі шляхи вирішення проблеми:
1. Можливо на приставці стояв AutoRCM або вона сіла - [найбільш розлога інструкція з запуску приставки, що сіла](https://4pda.to/forum/index.php?showtopic=900987&st=16320#entry89527321){:target="_blank"}
1. Спробуйте запустити {% include abbr/uk/hekate.txt abbr="hekate" %}. Якщо вийшло, можливо ви намагалися відкотити прошивку через що стався конфлікт між версією прошивки та спаленими ф'юзами. Спробуйте запустити **Full Stock** та [оновити прошивку до останньої версії](/update-fw){:target="_blank"}, або відключити Emunand (hekate > emuMMC > Change emummc > Disable) та оновити прошивку через налаштування
1. [Відновіть бекап](/backup-nand#відновлення-резервної-копії){:target="_blank"}, якщо такий є
1. Зробіть [відновлення прошивки](/downgrade_fw){:target="_blank"}

{% endspoiler %}

{% spoiler Після прокидання пейлоада та появи сплешскрінів, консоль висить у чорному екрані / на логотипі Atmosphere / на логотипі Nintendo. %}

1. Переконайтеся, що можете завантажитися в сток. Якщо це так, можливо ви встановили якісь системні модулі, які заважають завантаженню. Для цього:

{% spoiler Після запуску приставки помилка на фіолетовому фоні з написом **Error Code** та номером помилки %}

Зверніть увагу на числа у другому рядку після **Title:**. Ці числа вказують на те, який тайтл викликав помилку і через який не завантажується приставка. Якщо номер тайтла **не** `010000000000000D`, то перейдіть у папку `atmosphere/contents` (`atmosphere/titles` на атмосфері нижче ніж 0.10.0) і видаліть папку з номером, вказаним на екрані вашої приставки у другому рядку. Якщо номер тайтла - `010000000000000D`, то найімовірніше помилка викликана тим, що якесь хомбрю запущено в режимі аплету і йому не вистачає пам'яті.

### Проблеми з запуском ігор

{% spoiler При запуску ігор в emunand вилізає настирливе вікно про хмарні збереження %}

Якщо ви пропустили цей пункт і у вас були увімкнені хмарні збереження, то в емунанді при запуску ігор буде вилітати помилка, щоб її усунути, в **емунанді** через DBI видаліть акаунт на якому з'являється помилка через **Tools > Delete account**, після чого створіть новий з тим самим ім'ям. 

{% endspoiler %}

{% spoiler Не можу використовувати картриджі в EmuNAND, хоча в SysNAND все працює %}

Для коректної роботи картриджів SysNAND та EmuNAND повинні знаходитися на одній версії прошивки. Якщо це не так, [оновіть EmuNAND](/update-fw). Якщо це не допомогло, переконайтеся, що в hekate у вас відключено **NoGC** (**Options** -> **Auto NoGC** -> **OFF**)

{% endspoiler %}

{% spoiler Після запуску приставки помилка на чорному фоні з написом "**Panic occurred while running Atmosphere**". %}

Оновіть {% include abbr/uk/kefir_addr.txt %}. 
Якщо оновлення {% include abbr/uk/kefir_addr.txt %} не допомогло, зверніть увагу на числа у другому рядку після **Title:**. Ці числа вказують на те, який тайтл викликав помилку і через який не завантажується приставка. Якщо номер тайтла **не** `010000000000000D`, то перейдіть у папку `atmosphere/contents` (`atmosphere/titles` на атмосфері нижче ніж 0.10.0) і видаліть папку з номером, вказаним на екрані вашої приставки у другому рядку. Якщо номер тайтла - `010000000000000D`, то найімовірніше помилка викликана тим, що якесь хомбрю запущено в режимі аплету і йому не вистачає пам'яті. Якщо номер тайтла - `010000000000001F`, то витягніть картридж із приставки. Тримайте emunand та sysnand на одній версії прошивки, щоб надалі не було таких проблем.

![](/assets/images/switch/screenshots/atmos_title_error2.png)
{: .text-center}
{: .notice--info}

{% endspoiler %}

{% spoiler Хотів встановити гру, але встановлення зависло, гра в меню не з'явилася, але місце займає. Як видалити? %}

Запустіть **DBI** і виберіть "**Clean Orphaned Files**" - всі файли, на які немає посилань у системній базі даних, будуть видалені. **Важливо:** запускайте DBI тільки [в режимі аплету](/hbl){:target="_blank"}, кефір має бути свіжим. Інакше магія не спрацює. 
{% endspoiler %}

{% spoiler Не можу встановити ігри в NAND %}

Судячи з усього ви використовуєте EmuNAND і старий {% include abbr/uk/kefir_addr.txt %}. Оновіть його. 

{% endspoiler %}

{% spoiler В Tinfoil зникли всі ігри %}

* Переконайтеся, що на приставці стоїть вірна дата та час. 
* Оновіть {% include abbr/uk/kefir_addr.txt %} до актуальної версії 
* Видаліть tinfoil, оновіть {% include abbr/uk/kefir_addr.txt %}, встановіть tinfoil з папки `games` на карті пам'яті 
* [Ще поради](https://vk.com/topic-178659422_48077007){:target="_blank"}

{% endspoiler %}

{% spoiler Деякі ігри не запускаються з помилкою "Програма закрита, тому що сталася помилка" %}

Можливо, встановлений ReverseNX. Видаліть його. Також спробуйте вимкнути приставку, запустіть {% include abbr/uk/hekate.txt abbr="hekate" %}, перейдіть в **Payloads** і виберіть `fusee.bin`, після чого відразу затисніть одночасно обидві кнопки гучності доти, доки не потрапите в рекавері. Після вимкніть приставку утримуючи кнопку живлення. Потім увімкніть як зазвичай. 

{% endspoiler %}

{% spoiler Не запускається жодна гра. При запуску йде перевірка, потім помилка %}

Можливо, ви видалили тікети всіх ігор. Для перевстановлення тікетів, запустіть [tinfoil](/games){:target="_blank"}, перейдіть на вкладку "**Тікети**", і для кожної гри виконайте наступне:

1. Увімкніть опцію "**Встановити Ticket**"
1. Натисніть "**Почати встановлення**"

{% endspoiler %}

Для перевстановлення тікетів, запустіть [tinfoil](/games){:target="_blank"}, перейдіть на вкладку "**Тікети**", і для кожної гри виконайте наступне:

{% spoiler Після оновлення прошивки, деякі ігри перестали запускатися  %}

Найімовірніше, у вас перестали запускатися ігри, сконвертовані з xci через стару версію n4xci (ви могли завантажити вже сконвертований образ з будь-якого джерела; рідко коли джерело вказує, що гра - конвертка). Для вирішення цієї проблеми, скачайте свіжу версію [n4xci](https://github.com/The-4n/4NXCI/releases/tag/v4.03){:target="_blank"} (завантажуйте 4nxci-vX.XX**_GUI**.zip). Зробіть [дамп ключів на вашій актуальній прошивці](backup-nand#частина-iii---дампимо-ключі){:target="_blank"} і покладіть їх в одну папку з програмою. Не забудьте вказати шлях до них у полі **Keyset File**. Сконвертований образ NSP має почати працювати.

{% endspoiler %}

### Проблеми з файлами / картою пам'яті

{% spoiler Не можу закинути на карту гру більше 4х Гб, що робити?  %}

{% include /inc/fixatributes.txt %}

{% spoiler Приставка не бачить карту у форматі exFAT і просить оновити прошивку  %}

Його потрібно робити навіть у тому випадку, якщо у вас і так стоїть остання версія прошивки, що зламується! Повторне оновлення просто встановить відсутній драйвер. А взагалі, [переходьте на FAT32](http://format.customfw.xyz){:target="_blank"}!

{% spoiler Після запуску хомбрю помилка на фіолетовому фоні з написом Error Code та номером помилки %}

Ви запускаєте програму з режиму аплету (Applet mode), тобто з Альбомів, тоді як робити це потрібно в режимі тайла (Tile mode), тобто через гру або форвардер. Як правильно запускати HBL описано в [інструкції](/hbl){:target="_blank"}. Режим аплету не дає програмі, що запускається, повного доступу до всього обсягу оперативної пам'яті в приставці, від чого і трапляються помилки. Якщо номер тайтла - `010000000000000D`, то найімовірніше помилка викликана тим, що якесь хомбрю запущено в режимі аплету і йому не вистачає пам'яті. Якщо номер тайтла - `010000000000001F`, то витягніть картридж із приставки. Тримайте emunand та sysnand на одній версії прошивки, щоб надалі не було таких проблем. Також виправте атрибути (як шукайте на цій же сторінці)
{% endspoiler %}

{% spoiler Помилка при запуску хомбрю з головного меню приставки через форвардер %}

Вам потрібно оновити форвардер

{% endspoiler %}

### Проблеми з залізом

{% spoiler Джойкони в емунанді не працюють окремо від консолі, хоча в стоку сиснанда працюють %}

1. Синхронізуй joy-con'и з sysnand
1. Скиньте [Syncfixv2.te](/files/Syncfix.te){:target="_blank"} в корінь карти пам'яті
1. Скачайте пейлоад [TegraExplorer](https://github.com/suchmememanyskill/TegraExplorer/releases/){:target="_blank"} і помістіть його в папку `bootloader/payloads`
1. Встав карту в switch і зайди в hekate
1. Запусти `TegraExplorer.bin` з вкладки "**payloads**", якщо побачите "**Grabbing keys from prod.keys... Done***", натисніть будь-яку кнопку
1. Перейди за шляхом `[SD:/] SD CARD -> Syncfix.te -> Launch Script`
1. Після закінчення роботи скрипта, перезавантажте приставку 

{% endspoiler %}

**В:** [Вичерпна стаття](https://4pda.to/forum/index.php?showtopic=900987&st=14880#entry87254644){:target="_blank"}

### Проблеми з системою

{% spoiler У мене завантажилася прошивка, як її видалити не оновлюючись? %}
Запустіть **DBI** і виберіть "**Clean Orphaned Files**" - всі файли, на які немає посилань у системній базі даних, будуть видалені. **Важливо:** запускайте DBI тільки [в режимі аплету](/hbl){:target="_blank"}, кефір має бути свіжим. Інакше магія не спрацює. 

{% endspoiler %}

{% spoiler У мене остання прошивка, але при запуску деяких ігор я бачу, що "*доступне оновлення системи*". Чому так відбувається? %}

Мається на увазі оновлення гри. Скачайте його та встановіть як звичайну гру. Також скинути перевірку версії можна через DBI. Запустіть програму, виберіть **Browse installed applications**, натисніть {% include /inc/btn.txt btn="Y" %}, щоб виділити всі пункти, натисніть {% include /inc/btn.txt btn="A" %}, а потім виберіть **Reset required version**. DBI має бути [запущений з режиму аплету](/hbl){:target="_blank"}, кефір має бути свіжим. Інакше магія не спрацює. 

{% endspoiler %}

**DBI > Tools > Delete account**

{% endspoiler %}

{% spoiler Не синхронізується час %}

Автоматична синхронізація часу вимкнена в кефірі, починаючи з версії 660 для економії батареї в режимі сну. Для синхронізації використовуйте модуль [QuickNTP](https://github.com/nedex/QuickNTP){:target="_blank"} для Tesla (входить до складу кефіру). Запустіть Tesla, натиснувши одночасно {% include /inc/btn.txt btn="L" %} + {% include /inc/btn.txt btn="DOWN" %} + {% include /inc/btn.txt btn="R3" %}, виберіть QuickNTP, натисніть {% include /inc/btn.txt btn="A" %}. Якщо час не синхронізувався, виберіть інший сервер синхронізації хрестовиною (ВЛІВО-ВПРАВО).

{% endspoiler %}
